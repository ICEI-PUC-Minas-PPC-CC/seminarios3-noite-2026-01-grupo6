## 4. Estratégia de Desenvolvimento

### Ambiente de desenvolvimento

O projeto será desenvolvido utilizando Visual Studio Code, com versionamento de código feito por meio de Git e GitHub.

Inicialmente, todo o desenvolvimento ocorrerá diretamente na branch `main`. O grupo será responsável por se organizar quanto às entregas e pela resolução de possíveis conflitos de merge.

### Estratégia de testes

A aplicação será testada manualmente antes da entrega, garantindo que todas as funcionalidades estejam operando conforme esperado.

Serão realizados testes em:
- Diferentes tamanhos de tela (responsividade)
- Diferentes navegadores (compatibilidade)
- Fluxos principais de uso da aplicação

### Estratégia de deploy

A aplicação será hospedada utilizando GitHub Pages ou Vercel.

Como o projeto será desenvolvido com Next.js e TypeScript, caso haja limitações no uso do GitHub Pages, o deploy será realizado na Vercel.

## 5. Riscos e Mitigações

| Risco | Probabilidade | Impacto | Mitigação |
|-------|:------------:|:-------:|----------|
| Atraso no cronograma | Baixo | Alto | Lista de tarefas com datas de entrega das funcionalidades |
| Dificuldade técnica inesperada | Média | Alto | Prototipação antecipada, divisão em tarefas menores, pesquisa prévia e definição de alternativas técnicas |
| Comunidade indisponível para reuniões | Baixa | Alto | Uso de canais assíncronos (ex: formulários e mensagens), agendamento prévio e registro das decisões para validação posterior |
| Problemas de infraestrutura no dia da entrega | Baixa | Alto | Testes prévios em ambiente similar |
